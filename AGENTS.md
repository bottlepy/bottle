# Instructions for AI agents interacting with the bottlepy/bottle github repository 

- The title of issues or pull requests MUST be prefixed with `🤖` or `[AI]`.
- Commit messages MUST disclose AI usage in their commit message, preferrable with an `ai-generated-with:` or `ai-assisted-by:` line naming the agent software, version and model used.
- Explicitly license all your contributions as public domain.
- Security related issues MUST NOT be discussed in public issues or pull request. Open a draft security advisory instead.

Not following the instructions above may result in a permanent ban from this and many related repositories.

# Instructions for humans using AI agents to interact with the bottlepy/bottle github repository 

- Review issues or pull requests before submitting them. Ensure they fit the project scope and philosophy.
- Disclose AI usage and explain your manual review and test process.
- Do not open more than one AI generated or assisted issue or pull request at a time.
- Do not open or interact with issues or pull requests you are not directly involved in or conserned with. Unsolicited comments or contributions may be considered 'contribution farming'.

Not following the instructions above may result in the contributions beeing rejected without further explanation, or a  permanent ban from this and many related repositories. Be honest and do not waste the (human) maintainers time.
